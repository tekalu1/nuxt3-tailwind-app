## 🚀 開発フロー

### 基本的な実装フロー

```
1. Issue/タスク確認
   ↓
2. ブランチ作成（feature/xxx）
   ↓
3. 型定義の作成・更新（types/）
   ↓
4. サーバーAPI実装（server/api/）
   ↓
5. コンポーネント実装（components/）
   ↓
6. ページ実装（pages/）
   ↓
7. テスト作成・実行
   ↓
8. ドキュメント生成（pnpm docs:generate）
   ↓
9. コミット・プッシュ
   ↓
10. Pull Request作成
```

詳細は [Gitワークフロー](./guides/git-workflow.md) を参照。

## 📝 重要な原則

### 1. コードファースト開発

本プロジェクトは**コードファースト開発**を採用しています。

- **型安全性の重視**: TypeScript の型システムを最大限活用
- **自動化の推進**: ドキュメント生成、テスト、デプロイを自動化
- **コードの再利用**: Composition API と Composables で共通ロジックを抽出
- **段階的な実装**: 小さな単位で実装し、こまめにコミット

### 2. ドキュメント戦略

実装したコードから各種ドキュメントを自動生成することで、ドキュメントとコードの乖離を防ぎます。

詳細は [ドキュメント自動生成ガイド](../../docs-guide/generation-guide.md) を参照してください。

### 3. テスト駆動開発（推奨）

- 実装と同時にテストを作成
- 失敗するテストから書く（TDD）
- テストファイルのディレクトリ構造は実装ファイルと対応させる

詳細は [テストガイド](./guides/testing-guide.md) を参照してください。

## 🔍 実装前の確認事項

### チェックリスト

- [ ] 関連するドキュメントを読んだ
- [ ] 既存のコードスタイルを確認した
- [ ] 型定義を適切に記述した
- [ ] JSDocコメントを記述した（公開API・型定義）
- [ ] バリデーションを実装した（ユーザー入力がある場合）
- [ ] エラーハンドリングを実装した
- [ ] テストを作成した
- [ ] ドキュメント更新が必要か確認した
- [ ] ドキュメントを更新した（必要な場合）
- [ ] コミットメッセージが規約に従っている
