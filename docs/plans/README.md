## 計画作成プロセス

計画作成は以下の4フェーズで行います。各フェーズ完了時にレビューを実施します。

```
Phase 0: 初期ヒアリング（調査前）
    │   ├── ビジネス目的・背景の確認
    │   ├── スコープ・優先度の確認
    │   └── 受け入れ基準の確認
    │
    ↓  レビュー
    │
Phase A: 事前調査
    │   ├── 現状分析
    │   ├── 影響範囲の特定
    │   ├── 技術調査
    │   └── リスク特定
    │
    ↓
    │
Phase A': 確認ヒアリング（調査後）
    │   ├── 技術的アプローチの選択
    │   ├── スコープ調整の確認
    │   └── リスク対応方針の確認
    │
    ↓  レビュー
    │
Phase B: 計画書作成
    │   ├── タスク分割
    │   ├── フェーズ構成
    │   └── リスク対策まとめ
    │
    ↓  レビュー
    │
次工程（要件定義）へ
```

### ヒアリングの特徴

- **選択肢ベース**: ユーザーの負担を軽減するため、可能な限り選択肢を提示
- **汎用＋動的**: 汎用的な選択肢テンプレート＋プロジェクト固有の選択肢を動的生成
- **2段階**: 調査前（初期）と調査後（確認）の2回実施

## 使い方

### 新しい計画を作成する

1. `_template` フォルダをコピー
2. フォルダ名を計画名にリネーム
3. `hearing.md` でヒアリングを記録
4. `plan.md` で計画を作成

```bash
# 例: user-authentication という計画を作成
cp -r _template user-authentication
```

### 命名規則

- フォルダ名はケバブケースを使用
- 例: `user-authentication`, `api-redesign`, `performance-optimization`

## ディレクトリ構成

```
plans/
├── README.md                  # このファイル
├── _template/                 # テンプレート（コピー用）
│   ├── hearing.md             # ヒアリングテンプレート
│   └── plan.md                # 計画テンプレート
└── [計画名]/                  # 計画フォルダ（例: user-authentication）
    ├── hearing.md             # ヒアリング記録
    └── plan.md                # 計画書
```

## ステータス一覧

| ステータス | 説明 |
|-----------|------|
| ヒアリング中 | 初期ヒアリング実施中 |
| 調査中 | 事前調査実施中 |
| 未着手 | 計画作成済み、作業未開始 |

| 完了 | 全タスク完了 |
| 保留 | 一時停止中 |
| 中止 | 計画中止 |

## ヒアリング項目一覧

### Phase 0: 初期ヒアリング

| カテゴリ | 確認項目 | 選択肢例 |
|---------|---------|---------|
| ビジネス目的 | 主な目的 | 新規機能 / 改善 / バグ修正 / パフォーマンス / 技術的負債 / セキュリティ |
| ビジネス目的 | 変更理由 | ユーザー要望 / ビジネス要件 / 技術的必要性 / 競合対応 / 法規制対応 |
| スコープ | 規模感 | 小規模 / 中規模 / 大規模 |
| スコープ | 優先度 | 最優先 / 高 / 中 / 低 |
| スコープ | 含むもの | UI / ロジック / データ / API / テスト / ドキュメント |
| 受け入れ基準 | 完了基準 | 機能動作 / テストパス / レビュー承認 / パフォーマンス / ドキュメント |
| 受け入れ基準 | 品質特性 | 機能性 / 使いやすさ / パフォーマンス / 保守性 / セキュリティ / 拡張性 |

### Phase A': 確認ヒアリング

| カテゴリ | 確認項目 | 内容 |
|---------|---------|------|
| 技術選択 | アプローチ | 調査で発見された技術的選択肢から選択 |
| スコープ | 調整 | 調査で判明した追加・除外項目の確認 |
| リスク | 対応方針 | 許容 / 軽減策実施 / 回避（スコープ変更） |
